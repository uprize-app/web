# Uprize — Design Context

## Target audience

- **주 사용자**: 중소 규모 시행사 대표, 40~60대
- 건축 전문가 이전 단계. 건축사무소 방문 전 "이 필지에 어떤 건물이 어울릴까"를 혼자 시뮬레이션하고 싶어하는 의사결정자.
- 디지털 유창도는 중간. 모던하지만 과장된 "테크" 톤은 거부감. 미팅에서 공유해도 부끄럽지 않은 품격이 필요.

## Use cases / context

- **사무실 PC**: 필지 선택, 조건 입력, 결과 검토가 주 작업. 큰 화면 + 마우스.
- **현장 모바일**: 답사 중 사진 업로드, 결과 재확인, 외부 공유. 세로 화면 + 터치. 햇빛 아래 가독성 중요.
- 두 맥락 모두 지원해야 함. 모바일에서 기능을 삭제하지 말고 "레이아웃만" 바꿈.

## Brand tone

- **방향**: Editorial · Architectural monograph · 정제된 뼈색(bone) + 선명한 주황(#E85420) 포인트.
- 현재 `globals.css`의 팔레트/타이포를 **유지 강화**.
  - Neutrals: `#ECEAE4` (bone) / `#F5F3EE` (paper) / `#15140F` (ink)
  - Brand: `#E85420` (brand), `#C7461A` (brand-deep)
  - Display: Instrument Serif · Body: Space Grotesk · Mono: JetBrains Mono
- 무드: 건축 잡지, 부동산 IR 자료, 건축가 포트폴리오 사이 어딘가. 차분하고 단단하고 확신 있는.

## Differentiators — must avoid

- **AI 티 안 나게**. 보라/시안/그라디언트/글래스모피즘/네온 액센트 금지. 카드 남발 금지. 모든 heading 위에 아이콘 금지.
- **한글에 알맞은 화면 배치**. 한글 본문은 라틴 grotesque만으로 부족 — Pretendard 계열 한글 폰트가 반드시 필요. 대문자 letter-spacing 트릭(`letter-spacing: 0.08em; text-transform: uppercase`)은 **라틴 텍스트에만** 적용. 한글에는 금지.
- 한글 줄바꿈/어절 균형, 문장 길이, 숫자 혼용 시 정렬까지 신경 써야 "건축 에이전트"로 읽힘.

## Implementation constraints

- Next.js 16 App Router · React 19 · Tailwind v4 · shadcn/ui(base-ui)
- 기존 `src/app/globals.css`의 2,300+ 줄 수제 CSS는 **파괴적 리팩터 금지**. 토큰/타이포/접근성만 손봄.
- 한글 폰트는 SSR 안전하고 CLS 없이 로드. `next/font` 우선.
